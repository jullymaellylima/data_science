Titanic: Investigando o naufrágio através dos dados.¶

O desafio do Titanic ficou popularmente conhecido por conta do Kaggle, e considerado um dos passos iniciais na rea de machine learning.

Em 15 de abril de 1912, ao colidir com um iceberg, o Titanic naufragou, ocasionando a morte de 1502 pessoas - de um total de 2224.

Utilizando uma base de dados real, o objetivo desse desafio mensurar a chance de sobrevivencia dos passageiros do Titanic, de acordo com suas caractersticas.

Checklist do relatório:¶

  1. Definio do Problema;
  2. Obteno de Dados;
  3. Explorao de Dados;
  4. Preparao dos Dados;
  5. Construo do Modelo;
  6. Avaliao.

1. Definição do Problema¶

O naufrgio do Titanic um dos mais famosos da histria, e ocorreu na sua viagem inaugural, em abril de 1912, quando era considerado "inaufragvel". Com a ausncia de botes salva-vidas para todos, houve a morte de 1.502 pessoas entre os 2.224 passageiros e tripulantes.

Apesar do fator sorte envolvido na sobrevivncia, analisando os dados possvel perceber que determinados grupos de pessoas tinham mais chance de sobreviver que outros.

A descrio do desafio e o dataset esto disponveis na pgina do Kaggle: Titanic - Machine Learning from Disaster

Objetivo¶

Construo de um modelo preditivo, que utilize os dados dos passageiros para responder quais grupos de pessoas tm mais probabilidade de sobrevier.

Métrica de desempenho¶

O score calculado baseado no nmero de previses corretas do modelo, ou seja, considera a acurcia do modelo.

2. Obtenção dos Dados¶

O Kaggle disponibilizou os dados em dois grupos:

  • Dataset de treino - train.csv

    • Esse dataset informa se o passageiro sobreviveu ou no e deve ser usado para desenvolver o modelo de Machine Learning.
    • Possui diversas features, como classe, idade e gnero do passageiro.

  • Dataset de Teste - test.csv

    • Esse conjunto de dados no informa se o passageiro sobreviveu ou no e deve ser usado como dados que nunca foram vistos pelo modelo.

O Kaggle disponibiliza um modelo de como as previses devem ser submetidas plataforma (gender_submission)

3. Exploração de Dados¶

O estudo de cada varivel realizado, com ateno aos seus nomes, tipos, porcentagem de valores ausentes, outliers e tipos de distribuio.

O modelo utilizado classificado como de Aprendizado Supervisionado (Supervised Learning) e a varivel alvo (target) a varivel Survived.

Dicionário de Dados¶

  • PassengerId: Nmero de identificao do passageiro
  • Survived: Informa se o passageiro sobreviveu ao desastre
    • 0 = No
    • 1 = Sim
  • Pclass: Classe do bilhete
    • 1 = 1 Classe
    • 2 = 2 Classe
    • 3 = 3 Classe
  • Name: Nome do passageiro
  • Sex: Sexo do passageiro
  • Age: Idade do passageiro
  • SibSp: Quantidade de cnjuges e irmos a bordo
  • Parch: Quantidade de pais e filhos a bordo
  • Ticket: Nmero da passagem
  • Fare: Preo da Passagem
  • Cabin: Nmero da cabine do passageiro
  • Embarked: Porto no qual o passageiro embarcou
    • C = Cherbourg
    • Q = Queenstown
    • S = Southampton

Valores ausentes¶

Analisados os valores que faltam em cada coluna, verificado qual aborgem cabe a situao. Existem duas mais populares, que so preencher os valores arbitrariamente - com a mdia, mediana ou moda - ou excluir a linha inteira. H tambm a possibilidade de aplicao de um modelo de machine learning para preencher os dados ausentes.

Cabe ao responsvel verificar o que melhor para cada anlise, mas, na maioria das vezes, no recomendvel excluir uma linha inteira por conta de um valor ausente, e sim preencher o campo.

Preparação das variáveis para o modelo¶

necessrio transformar variveis categricas em variveis numricas, por conta do modelo matemtico. Ento, a varivel Sex convertida para male:0, e female:1. Na varivel Embarked aplicado o conceito de variveis dummies, onde uma nova coluna criada para cada novo valor da varivel, e assumem apenas os valores 0 e 1.

5 Construção do Modelo e Avaliação¶

Ser construdo um modelo bsicos d Machine Learning, o Modelo de Regresso Logstica.

In [20]:
#importar biblioteca do modelo de Machine Learning

from sklearn.linear_model import LogisticRegression

#criar o modelo de Regresso Logstica

lr_model = LogisticRegression(solver='liblinear')
lr_model.fit(train, target)

#verificar a acurcia do modelo
acc_logReg = round(lr_model.score(train, target) * 100, 2)
print(f"Acurcia do Modelo de Regresso Logstica: {(acc_logReg)}")

Acurcia do Modelo de Regresso Logstica: 80.13

Utilizando os dados de treino, o modelo de Regresso Logstica criado apresentou acurcia acima de 80%. Agora, o modelo ser aplicado em cima dos dados de teste.
